我們正在做什麼 Current Initiative

我們已經開始使用 AI 編程工具（Claude Code、Gemini CLI 等）透過 SSH 連線到伺服器，以 WP-CLI 指令遠端管理五個 WordPress 站（TW / APAC / EU / JP / IN）。 We are now leveraging AI-powered tools (Claude Code, Gemini CLI) to remotely operate all five regional WordPress sites (TW / APAC / EU / JP / IN) through SSH and WP-CLI. This eliminates the need to manually log into each admin panel, enabling centralized, efficient management from a single terminal.

適用範圍聲明 Scope of Access

本方案僅涉及亞太區域管理的五個站點（TW / APAC / EU / JP / IN），這些站點部署在獨立的伺服器環境中。美國官網（engeniustech.com 主站）不在本方案的存取範圍內，SSH 連線、WP-CLI 指令及 Cloudflare Tunnel 配置均不會觸及美國站的伺服器、資料庫或任何資源。 This initiative is strictly scoped to the five APAC-managed regional sites (TW / APAC / EU / JP / IN), which are hosted on a separate server environment. The US site (engeniustech.com main) is entirely outside the scope of this access. No SSH connections, WP-CLI commands, or Cloudflare Tunnel configurations will touch the US site’s server, database, or any associated resources.

為什麼需要 Cloudflare Tunnel The Security Gap -- Why Cloudflare Tunnel Is Required

這套 AI 遠端管理會是長期且頻繁的操作。但目前的 SSH 連線方式有安全隱患： This AI-driven workflow is already delivering measurable ROI and will scale further. However, the current SSH configuration bypasses Cloudflare’s protection layer, exposing our origin server. This is the gap Cloudflare Tunnel closes:

項目 目前（直連） Cloudflare Tunnel
主機 IP 暴露在公網 完全隱藏
防火牆 Port 需開放 18765 不需開任何 Port
連線安全 僅 SSH 加密 SSH + Cloudflare 雙層加密
存取控制 僅 SSH Key 可加 Cloudflare Access 身份驗證
DDoS 防護 無 Cloudflare 全球網路防護
Item Current (Direct) Cloudflare Tunnel
Server IP Exposed publicly Completely hidden
Firewall Port Must open port 18765 No ports needed
Connection Security SSH encryption only SSH + Cloudflare dual encryption
Access Control SSH Key only Can add Cloudflare Access authentication
DDoS Protection None Cloudflare global network protection

Cloudflare Tunnel 讓我們安全地持續使用 AI 管理工具，而不用在防火牆上開後門。 Cloudflare Tunnel enables us to maintain full AI-powered operational capability with zero-trust security -- no exposed ports, no visible origin IP, no firewall exceptions.
