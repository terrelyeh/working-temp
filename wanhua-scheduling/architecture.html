1. 系統目標

取代目前的 Google Sheet 排班流程，解決四件事：

  • 排班省時：主管手動排班時自動檢查違規
  • 員工查班方便：手機 PWA 隨時查個人班 / 全店班
  • 減少排班錯誤：工時達標、關鍵時段、11h 休息等自動提醒
  • 計算工時：自動計算目標工時、實排、補時數（不算薪資）

2. 使用者角色與權限

角色成員主要權限
高階主管 Winnie / Fish / 政道 Winnie：排班、所有操作
Fish / 政道：🟡 確認是否僅查看
正職 超威、蛋頭、思坦 填休假表、查詢班表、提出換班申請
PT 人力 pool（🟡 完整名單待補） 填排班意願表、查詢班表
登入方式：Google Login（所有員工一個 Gmail）。

3. MVP 分階段範圍

三階段推進，每階段可獨立上線。Phase 1 是取代 Google Sheet 的核心。

Phase 1 電子化基礎（1–2 個月）

  • 員工登入（Google Login）
  • 主管建立月度行事曆（國假輸入 / 匯入人事行政局資料）
  • 正職填月休假表（取代 Google Doc）
  • PT 填月排班意願表
  • 系統自動計算正職目標工時
  • 主管手動拖拉排班 + 即時違規檢查（紅燈 / 黃燈）
  • 員工 PWA 查個人班表 + 全店班表
  • 班表發布 + 個人班表分享連結

Phase 2 流程完善（+1 個月）

  • 換班申請（員工申請 → 主管核准）
  • 變更 log（誰在什麼時候改了什麼）
  • 月報 / 統計（工時達成率、晚班次數、加班、公平性指標）
  • 打卡 vs. 排班比對（🟡 需確認是否有打卡系統）

Phase 3 自動化（+2 個月）

  • 自動排班建議（演算法給 1–3 個可行解，主管選一個微調）
  • Excel / PDF 匯出
  • 補時數自動追蹤（跨月計算）

4. 核心業務流程

4.1 月排班週期

  1. 月底前第 N 天：主管公告下月行事曆（國定假日、補班日）
  2. 正職填休假表：固定週休 + 額外休假 + 特休 + 特殊協議
  3. PT 填排班意願表：可排 / 不可排的日期與時段
  4. 系統計算正職目標工時（見 4.2）
  5. 主管排班：手動拖拉 + 違規檢查
  6. 發布：月底最後一週完成，班表上線
  7. 月中異動：請假、換班、臨時調整走變更流程

4.2 正職目標工時公式

「目標工時」有兩個含義，系統主要使用的是「排班目標工時」：
  • 月工時總額：員工該月對公司的總貢獻（含特休、會議）
  • 排班目標工時（系統核心）：實際要排進班表的時數，= 月工時總額 − 特休時數 − 會議時數
月工時總額   = (當月總日數 − 固定週休 − 特殊協議 − 國假) × 8H

排班目標工時 = 月工時總額 − 特休時數 − 會議時數   ← 系統實際用這個
總達成判定   = 實排時數 ≥ 排班目標工時
🟡 核心公式待 Winnie 確認

依 4 月超威（152H / 134H + 16H 特休 + 2H 開會）反推，152H 可能就是「排班目標工時」。但推算的「月工時總額」168H（= 21 天 × 8）vs. 152H 差 16H 剛好等於特休，公式是否為「月工時總額不含特休天數」？請 Winnie 確認正確公式。

4.3 休假衝突處理

  • 除週一公休外，同日休假正職人數有上限（🟡 數字待定，原「≤ 3 人」因只剩 3 位正職已失效）
  • 衝突時：早填早優先

4.4 班表變更流程

  • 只有 Winnie 能改班表
  • 每次變更留 log：時間、操作者、變更前後、原因
  • 影響員工 → 通知（Email / LINE / 站內）

4.5 換班流程

  • 員工 A 提出想與員工 B 換班（指定日期 / 班次）
  • 員工 B 同意
  • Winnie 核准 → 系統自動更新班表
  • 過程留 log

5. 資料模型

5.1 核心資料表

employees
  id, name, email, role (高階主管/正職/PT),
  active, joined_at, notes

employee_settings  -- 正職專用
  employee_id,
  fixed_rest_days: [0..6]  -- 0=Sun, 1=Mon, ...
  can_work_10h: bool
  annual_leave_quota: int  -- 年假天數
  annual_leave_expires_at: date

shifts  -- 班次主檔
  code, name, start_time, end_time,
  nominal_hours, actual_hours,
  day_type: ['weekday', 'long_day', 'sunday', 'all']

monthly_calendars
  year_month,
  holidays: [date],
  makeup_days: [date],
  published: bool

leave_requests  -- 正職月休假表
  employee_id, year_month,
  dates: [date],          -- 額外休假日期
  special_agreement_days: int  -- 例：超威 +1
  annual_leave_hours: int,      -- 特休時數
  submitted_at

pt_availability  -- PT 月排班意願
  employee_id, year_month,
  available_slots: [{ date, shifts[] }]
  submitted_at

monthly_targets  -- 正職每月目標（系統計算）
  employee_id, year_month,
  work_days, base_hours,
  meeting_hours,          -- 彈性填入
  target_hours_for_schedule,  -- 實際要排進班表的
  carryover_hours,        -- 上月欠 / 超
  computed_at

schedules  -- 班表
  id, date, employee_id, shift_code,
  published: bool,
  is_important: bool,     -- 重要事項標記
  notes: text             -- 班內備註（不納入系統但保留欄位）

schedule_changes
  schedule_id, changed_by, changed_at,
  before_json, after_json, reason

swap_requests
  from_employee, to_employee,
  dates: [{ from_date, to_date }],
  status: pending/approved/rejected,
  approved_by, approved_at

5.2 計算邏輯（放 Server 不放 Client）

  • 目標工時計算：monthly_targets 在休假表提交後自動寫入
  • 違規檢查：排班時即時跑（工時、11h 休息、連續天數、關鍵時段）
  • 達成判定：每次排班異動重算
